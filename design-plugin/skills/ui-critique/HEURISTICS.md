# UI評価フレームワーク

Nielsenのヒューリスティクス、WCAG、プラットフォームガイドラインに基づく評価基準。

---

## Nielsenの10のユーザビリティ・ヒューリスティクス

### #1 システム状態の可視性

- ユーザーは現在何が起きているかわかるか？
- ローディング状態が明示されているか？
- 進捗インジケータがあるか？

### #2 システムと現実世界の一致

- 専門用語ではなくユーザーの言葉を使っているか？
- アイコンは直感的か？
- メタファーは適切か？

### #3 ユーザーの制御と自由

- 「戻る」「キャンセル」「やり直し」が可能か？
- 誤操作からの復帰が容易か？

### #4 一貫性と標準 ⭐視覚評価で重要

- プラットフォーム標準に従っているか？
- 独自アイコンでユーザーを混乱させていないか？
- 同じ機能に異なるデザインを使っていないか？

**チェックポイント**:
- [ ] ボタンスタイルの統一
- [ ] 色の意味の一貫性（赤=エラー等）
- [ ] アイコンスタイルの統一（Outline/Filled）
- [ ] 余白の規則性（8ptグリッド）

### #5 エラー防止

- エラーを起こしにくい設計か？
- 破壊的アクション前に確認があるか？

### #6 記憶負荷より認識

- 情報は見て分かるか（覚える必要がないか）？
- 選択肢は可視化されているか？

### #7 柔軟性と効率性

- ショートカットがあるか？
- パワーユーザーに配慮しているか？

### #8 美的で最小限のデザイン ⭐視覚評価で重要

- 無関係な情報や装飾過多（うるさい）ではないか？
- ノイズ対シグナル比は適切か？
- 余白は十分か？

**チェックポイント**:
- [ ] 不要な装飾を削除
- [ ] 余白の確保（最低16px）
- [ ] 強調要素は1-2個に絞る
- [ ] 情報の優先順位付け

### #9 エラーからの回復支援

- エラーメッセージは明確か？
- 解決策を提示しているか？

### #10 ヘルプとドキュメント

- 必要時にヘルプにアクセスできるか？

## アクセシビリティ（WCAG 2.1）

### コントラスト比

| 対象 | 最低基準（AA） | 推奨基準（AAA） |
|------|---------------|----------------|
| 通常テキスト（<18pt） | **4.5:1** | 7:1 |
| 大型テキスト（≥18pt） | **3.0:1** | 4.5:1 |
| UIコンポーネント | **3.0:1** | - |

### 色だけに依存しない

- エラーを赤色だけで示していないか？
- 状態変化に色以外の手がかりがあるか？（アイコン、テキスト、下線等）

### タッチターゲット

| プラットフォーム | 最小サイズ |
|-----------------|-----------|
| iOS (HIG) | 44pt × 44pt |
| Android (Material) | 48dp × 48dp |

## プラットフォームガイドライン

### Apple HIG チェック

- [ ] タップ領域は44pt以上か？
- [ ] テキストサイズは11pt以上か？
- [ ] Safe Areaを考慮しているか？
- [ ] Dynamic Typeに対応しているか？
- [ ] セマンティックカラーを使用しているか？

### Material Design チェック

- [ ] 8dpグリッドに従っているか？
- [ ] Elevation（立体感）は物理法則と矛盾しないか？
- [ ] 重要アクションは画面下部にあるか？
- [ ] FABは1画面1個以下か？

## 致命的エラーパターン（即時修正必須）

以下は最優先で指摘すること:

### レイアウト崩壊

| パターン | 説明 |
|---------|------|
| はみ出し | テキスト/要素が親コンテナからはみ出している |
| 重なり | 要素同士が意図せず重なっている |
| 切れ | テキストや画像が途中で切れている（ellipsisなし） |
| 空白崩壊 | 意図しない巨大な空白、または余白がゼロ |

### コンテンツエラー

| パターン | 説明 |
|---------|------|
| 文言重複 | 同じテキストが複数箇所に表示 |
| プレースホルダー残存 | "Lorem ipsum"、"テスト"、"XXX"等が残存 |
| データ不整合 | 同じ情報が異なる値で表示 |
| 言語混在 | 意図しない言語の混在（未翻訳） |

### インタラクション不整合

| パターン | 説明 |
|---------|------|
| 状態の矛盾 | 選択中なのに非選択スタイル |
| フィードバック欠如 | ボタンにタップ状態がない |
| ナビゲーション断絶 | 戻る手段がない、行き止まり |

|------|------|--------|

### 要改善（優先度高）
| 箇所 | 問題 | 関連原則 | 修正案 |
|------|------|----------|--------|

### 推奨改善（優先度中）
| 箇所 | 問題 | 関連原則 | 修正案 |
|------|------|----------|--------|

### 良い点
- [維持すべきデザイン要素]
