# UIデザイン批評スキル

「なんとなくダサい」「どこか違和感がある」を、具体的で修正可能な指摘に変換する。

## 批評の原則

**残酷なまでに正直に**: お世辞は不要。成長のために欠点を徹底的に言語化する。

## 批評ワークフロー

### ステップ1: 観察（Observation）

UI要素を客観的にリストアップ:
- ヘッダー、ナビゲーション、コンテンツ、ボタン等
- 各要素の色、フォント、配置を記述

### ステップ2: 分析（Analysis）

デザイン原則に照らして問題を特定:
- ゲシュタルト原則（近接、整列、類似性）
- Nielsenのヒューリスティクス
- WCAG アクセシビリティ基準

### ステップ3: 批評（Critique）

日本語デザイン批評用語で問題を言語化。詳細は[VOCABULARY.md](VOCABULARY.md)参照。

## 主要な批評用語

| 用語 | 検出する視覚的特徴 |
|------|-------------------|
| 目が滑る | コントラスト不足、視覚的階層の欠如 |
| 目に痛い | 高彩度色の隣接、補色のハレーション |
| うるさい | 余白不足、強調要素の乱立 |
| 違和感 | 一貫性の欠如、デザインシステムからの逸脱 |
| 垢抜けない | デフォルト設定、過剰な装飾、時代遅れの効果 |

## 出力形式

Markdownテーブルで出力:

```markdown
| UI要素 | 指摘内容 | 関連原則 | 重大度 | 修正案 |
|--------|----------|----------|--------|--------|
| ヘッダー | ロゴとナビの整列ズレ（垢抜けない） | ゲシュタルト（整列） | 高 | Flexboxでalign-items: center |
```

## 評価フレームワーク

詳細な評価基準は[HEURISTICS.md](HEURISTICS.md)参照。

## クイックチェック

1. **第一印象**: 信頼できる？安っぽい？
2. **視覚階層**: どこを見ればいい？
3. **一貫性**: 角丸、マージン、色は統一？
4. **余白**: 窮屈？うるさい？
5. **コントラスト**: 読みやすい？目に痛い？
