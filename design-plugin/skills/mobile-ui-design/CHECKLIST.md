# プロフェッショナルUIデザイン品質チェックリスト

デザインレビュー・実装前の最終確認に使用。

---

## 1. タイポグラフィ

### 基本サイズ
- [ ] 本文サイズは **16px (16sp/17pt) 以上**か？
- [ ] 14px未満の重要情報はないか？
- [ ] 10px未満のテキストはLegal/免責のみか？

### 行間 (Line Height)
- [ ] 欧文の行間は **1.2〜1.5倍** か？
- [ ] 和文の行間は **1.5〜1.8倍** か？
- [ ] 長文で行間が狭すぎて窮屈に見えないか？

### ジャンプ率
- [ ] 見出しと本文のサイズ差は明確か？
- [ ] 見出しは本文の **1.5〜2倍以上** か？
- [ ] 視覚的な階層構造が一目でわかるか？

### 和欧混植
- [ ] 漢字が欧文より大きく見えないか？
- [ ] 必要に応じ和文を90-95%に縮小しているか？

### アクセシビリティ
- [ ] 固定pxではなく **Dynamic Type / sp単位** で実装されているか？
- [ ] ユーザーがシステム設定で文字サイズを変更した場合、レイアウトは破綻しないか？

---

## 2. レイアウト＆スペーシング

### 8ptグリッド
- [ ] 全ての余白は **8の倍数（8, 16, 24, 32...）** か？
- [ ] コンポーネントの高さは8の倍数か？（40px, 48px, 56px等）
- [ ] 中途半端な数値（14px, 18px, 25px等）を使っていないか？

### タッチターゲット
- [ ] タップ可能要素は **44px × 44pt（iOS）/ 48dp（Android）以上** か？
- [ ] 視覚的に小さいアイコンも、Hit Areaは確保されているか？
- [ ] 隣接する要素との間隔は十分か？（誤タップ防止）

### グルーピング
- [ ] **内部余白 < 外部余白** のルールが守られているか？
- [ ] 関連する要素は近接しているか？
- [ ] 無関係な要素は十分に離れているか？

### マージン
- [ ] 画面端の余白は **16-24dp** 確保されているか？
- [ ] コンテンツが画面端にぴったり張り付いていないか？

---

## 3. 視覚品質（脱「ダサい」）

### 純黒回避
- [ ] **#000000** を使用していないか？
- [ ] ダーク背景は **#121212** 等のダークグレーを使用しているか？
- [ ] テキストカラーにも純黒を避けているか？

### シャドウ
- [ ] デフォルトシャドウ設定を避けているか？
- [ ] **Blur値はY軸の2〜4倍** に設定しているか？
- [ ] シャドウの不透明度は **10-15%** 程度か？
- [ ] 必要に応じて多重レイヤーシャドウを使用しているか？

### 角丸計算
- [ ] ネストされた角丸は **外側 = 内側 + パディング** の式に合致しているか？
- [ ] 角丸の厚みが均一に見えるか？

### 整列
- [ ] 長文テキストは **左揃え** か？
- [ ] 安易な中央揃えをしていないか？
- [ ] 両端揃えを使用していないか？
- [ ] 見えないグリッド線で要素が整列しているか？

### アイコン
- [ ] 線画(Outline)と塗り(Filled)のスタイル混在を避けているか？
- [ ] 全アイコンの **線の太さは統一** されているか？
- [ ] 角の形状は統一されているか？

---

## 4. カラー＆ダークモード

### 配色比率
- [ ] **ベース60% / メイン30% / アクセント10%** のバランスになっているか？
- [ ] アクセントカラーは使いすぎていないか？
- [ ] ブランドカラーが適切に反映されているか？

### ダークモード深度
- [ ] ダークモード時、高い要素ほど **明るいグレー** で表現されているか？
- [ ] 影ではなく表面色で深度を表現しているか？
- [ ] Elevation階層が視覚的に明確か？

### コントラスト
- [ ] 通常テキストのコントラスト比は **4.5:1以上** か？
- [ ] 大型テキスト/アイコンは **3.0:1以上** か？
- [ ] ダークモードで高彩度色を避けているか？
- [ ] 色覚多様性に配慮しているか？（色だけで情報を伝えていないか）

---

## 5. インタラクション

### フィードバック
- [ ] タップ/クリック時に視覚的フィードバックがあるか？
- [ ] ローディング状態が明示されているか？
- [ ] エラー状態が明確に表示されるか？

### 状態表現
- [ ] ボタンのEnabled/Disabled状態が区別できるか？
- [ ] 選択状態が視覚的に明確か？
- [ ] ホバー状態（タブレット/デスクトップ）が設計されているか？

---

## 6. プラットフォーム準拠

### iOS (HIG)
- [ ] SF Symbolsまたは互換アイコンを使用しているか？
- [ ] iOSのナビゲーションパターンに準拠しているか？
- [ ] Safe Areaが考慮されているか？

### Android (Material)
- [ ] Material Design 3のガイドラインに準拠しているか？
- [ ] システムナビゲーション（ジェスチャー/3ボタン）に対応しているか？
- [ ] Edge-to-edgeレイアウトに対応しているか？

## クイックサマリー

| カテゴリ | 最重要チェック |
|---------|---------------|
| タイポグラフィ | 本文16px以上、和文行間1.5倍以上 |
| レイアウト | 8ptグリッド、タップ44-48px以上 |
| 視覚品質 | 純黒禁止、シャドウBlur=Y×2以上 |
| 角丸 | 外側 = 内側 + パディング |
| 配色 | 60-30-10、コントラスト4.5:1以上 |
| ダークモード | 高さで明るさ、影ではなく表面色 |
